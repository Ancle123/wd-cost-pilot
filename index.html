1
三件核心工作的价值定位
按涂峰总要求：以创造价值大小排序
01
价值创造核心
采购管理 + 供方管理
创造价值最大。通过供方分层管理、招采规划、方案优化实现降本。

关键动作：
- 建大区供方功能库（当前缺失）
- 供方分层：保洁属地化、电梯空调全国化
- 避免主观切换供方
- 招采规划前置，减少频繁招标
02
价值保障重要
验收和核价
保障价值不流失。验收是供方履约质量的最后防线，核价为业务决策提供依据。

关键动作：
- 验收分级：重大项目必须现场验收
- 小型项目可不验收，释放人力
- 核价聚焦辅助决策，减少非必要投入
- 建价格库，库内项目自动通过
03
价值闭环支撑
交付后验收结算
完成价值闭环。结算审核确保成本准确归集，防止超付。

关键动作：
- 属地初验+大区终审
- 业务部门不发起则结算无法推进
- 现场发现问题可直接扣减（如吴胶厂扣20万）
- 数据反馈支撑下一轮采购决策
涂峰总明确指出的当前问题
- 测算精力投入过大：上海吴中路项目测算已第12版，被动配合资方要求不断调整
- 验收放弃影响严重：因人员减少、差旅费等原因放弃验收，导致供方以次充好（如保温层厚度问题）
- 中台势能未发挥：大区成本人员精力被牵制，招采系统不稳定、无法分标段招标、缺乏供方库
- 供方管理失控：供方库因安全事故等原因放弃，鱼龙混杂、挂靠现象多，成本对供方缺乏约束能力
- 定位偏差：以往定位为「服务业务部门」，导致频繁招标、供方更换频繁（平均存活不超一年）

调整项当前做法调整后做法价值

集采判断单一标准看集采集采是否降本取决于采购需求和供应商市场环境，非单一标准保障
2. 验收和核价（价值保障）
调整项当前做法调整后做法价值
验收执行因人员减少/差旅费放弃验收属地验收专员常驻城市，现场验收不再受差旅费限制关键
验收分级无差异化标准按项目规模分级：大型施工必验收、小型可不验收提效
核价定位测算精力投入大（吴中路12版）核价为辅助决策，减少非必要投入，聚焦关键项提效

调整项当前做法调整后做法价值
结算发起业务部门不发起则无法推进属地成本组长督促发起，建立结算计划台账闭环

问题处理发现问题需上报中台反馈属地现场发现问题直接处理，扣减有据可查闭环
数据反馈数据建设滞后结算数据反哺供方评估库和采购决策支撑

合同全周期寿命管理（领导认同）
涂峰总认同合同全周期寿命管理理念，以下职责梳理按合同生命周期展开：从需求发起 -> 招采规划 -> 供方选择 -> 合同签订 -> 履约监督 -> 验收结算 -> 数据反馈。
需求发起广场端
招采规划大区
供方选择大区+属地
履约监督属地
验收结算属地+大区

单店（广场端）成本职责

  • 核价：单店日常零星核价（价格库内自动通过）
  • 批流程：广场端审批流程发起与跟进
  • 单店沟通：与广场各部门日常沟通协调
  • 需求发起：招采计划、结算计划、验收计划发起
  • 现场执行：现场施工监督、材料进场确认
  • 验收确认：业务部门验收确认参与
  • 数据反馈：广场端成本数据录入与反馈
  • 日常对接：与城市公司成本组长日常对接

城市公司成本职责

  • 合价测算：合价测算组织与初步审核
  • 现场勘察：测算项目现场实际情况勘察
  • 例会参与：城市公司各级例会（月度/周度）
  • 层级对接：城市公司各层级（总经理/工程物业部/营运部）对接
  • 大区汇报：向大区小中台汇报、承接大区指令
  • 供方对接：属地供方日常对接与履约监督
  • 结算初验：属地结算初验与资料归集
  • 督促发起：督促广场端发起结算/验收
  • 审批流转：属地审批权限内的变更签证审批

大区小中台职责

  • 测算核价组：复杂项目算量核价、单价审核
  • 价格库建设：价格基准库建设与维护
  • 招采规划：年度招采计划编制与组织
  • 供方管理：供方功能库建设、准入评估、分层
  • 集采执行：集采/区域采组织与执行
  • 方案优化：采购前置参与方案设计
  • 结算终审：大区层面结算终审与费用汇总
  • 数据分析：成本数据归集、分析与对标
  • 专业支持：对属地人员专业支持与带教
测算现场勘察的属地化解决方案
当前测算需要到现场勘察实际情况，但小中台化后无法每个项目都派人去现场。解决方案：按项目复杂度分级处理。

简单/标准项目

价格库内覆盖的项目
属地验收专员现场拍照记录
大区远程核价
无需派人到现场

中等复杂项目

库外但有参考基准
属地组长现场勘察+影像
大区核价组远程复核
关键节点大区抽检

复杂/大额项目

10万以上或无参考基准
大区核价组派人到现场
与属地组长联合勘察
确保测算准确性

原则：大区核价组不再对所有项目到现场，而是聚焦复杂/大额/库外项目。简单项目由属地验收专员完成现场记录，影像资料+价格库支撑大区远程核价。这样既保证测算质量，又解决人力不足无法每项目到现场的问题。

价格库与供方库：两项优先建设
解决核价工作量过大 + 供方管理失控的基础设施
为什么优先建？
人员已减少几个，核价工作量却非常大。如果不建价格库，每个人还是从零开始核每一项价格，中台16人根本扛不住。价格库建成后，库内项目自动通过，核价人力聚焦在库外项目和新材料新工艺上，才能在人手减少的情况下维持核价质量。
1. 价格库建设方案
阶段一：核心品类价格库
（保洁/绿化/石材/维保等高频品类）
->
阶段二：全品类扩展
（工程类/营销活动类/行政采购类）
->
阶段三：动态更新机制
（季度更新+市场价触发更新）

审核流程设计：属地2人后的管理难点
核心问题：城市公司只剩2人，审核工作怎么组织？
管理难点分析
当前每个城市公司有组长+若干下属，组长让下属复核后再上报。小中台化后城市公司只剩2人，验收专员聚焦现场验收，组长一个人扛属地所有成本审核+对接+结算初验。如果大区小中台还要按城市维度逐个审核属地提交的作业，工作量确实非常大。

核心矛盾：审核工作到底是按城市维度组织，还是按业务量/业务类型组织？
解决方案：按业务类型审核，非按城市审核
属地化前（按城市审核）

- 城市A所有事项 -> 城市A组长安排下属复核 -> 上报中台
- 中台按城市分对接人，每人盯几个城市
- 问题：中台对某城市业务全包，忙闲不均

属地化后（按业务类型审核）

- 7城验收事项 -> 各城验收专员直接处理
- 7城核价事项 -> 价格库内自动通过，库外的进入测算核价组
- 7城采购/供方事项 -> 进入采购与供方管理组
- 两组按业务量分配组员，不按城市分配
- 两组组长负责整体协调和异常升级，不参与逐项审核

关键设计：
1. 价格库是第一道闸口：库内项目自动通过，不进入人工审核队列
2. 属地组长做初筛：属地提交资料经组长初筛分类后，按业务类型推送至对应组
3. 两组按业务量配人：核价组4个工程师按业务类型分工（如A负责维保类、B负责工程改造类），任何人都能审任何城市的件
4. 两组组长角色定位：不审核具体件，负责整体进度监控、异常升级、跨组协调
5. SLA机制：核价2个工作日、结算终审5个工作日，超时自动升级

优势与风险
全面评估试点利弊

优势

  • 人员集中提高效率：改变工作方式，集中办公减少沟通损耗
  • 验收不再受差旅费限制：验收专员常驻城市
  • 供方履约监督常态化：组长+验收专员双人在场
  • 专业分工更清晰：两组各司其职，采购与供方管理创造直接价值
  • 价格库提效：库内自动通过，核价人力聚焦复杂项
  • 数据沉淀加速：集中后数据归集更快，反哺采购决策
  • 合同全周期管理：领导认同，从需求到数据反馈全链条覆盖
  • 复合用工灵活：两组可互援，应对业务高峰

风险

  • 人员集中导致离职：调至大区（上海）生活成本高
  • 新招人上手慢：新人对万达体系不了解，磨合期影响效率
  • 属地2人扛不住：组长一人兼审核+对接+结算初验
  • 测算无法每项目到现场：大区人力有限，需分级处理
  • 供方库建设滞后：功能库需时间，短期内供方管理仍处真空
  • 城市公司协同摩擦：双线管理可能有冲突
  • 保密风险：调整过程泄露可能引起恐慌
